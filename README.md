# 盛田教研｜國中落點地圖

這是一個可部署到 GitHub Pages 的靜態網站，用地圖呈現台中高中職落點資料。

## 功能規格

### 地圖範圍與標記

- 地圖以台中市為主要顯示範圍，外縣市區域以灰色遮罩弱化。
- 地圖底圖需保留路名與地名，方便家長辨識學校位置。
- 學校標記依最低錄取積點呈現不同顏色。
- 盛田教研位置固定顯示於地圖上，地址為「臺中市北區英才路112號」。
- 盛田教研標記使用藍色圓點，標記文字為「盛」，用來和學校標記區分。
- 點擊學校標記時，地圖維持使用者當下的位置與縮放，不自動縮回台中。
- 點擊地圖空白處時，已開啟的學校資訊會收合。

### 學校資訊

- 網站只呈現台中高中職資料。
- 同一所學校、同一個科系的資料需整理在一起。
- 學校資訊面板右上角提供關閉按鈕。
- 學校資訊面板不顯示「編輯資料」按鈕。
- 學校資訊中的學生資料若為特殊身分，需在資料前方加上 `★`。
- 學校資訊中的學生資料若同科系積點落差較大，需在資料前方加上 `△`。
- 標記說明需顯示在頁面中：
  - `★`：特殊身分資料，例如低收入戶、原住民等。
  - `△`：同科系積點落差較大的資料，可作為保守參考。

### 成績與篩選

- 頁面上方提供「我的成績」區塊，讓家長輸入國文、英文、數學、自然、社會與作文成績。
- 五科等級依 A、B、C 統計成「幾 A 幾 B 幾 C」。
- A、B、C 篩選條件加總必須等於 5，避免輸入不完整或不合理的組合。
- 提供積點篩選，可依最低積點與最高積點篩選學校。
- 篩選功能需集中在同一個區塊，並和學校清單區塊明確分開。
- 左側學校卡片、統計卡片、搜尋與篩選區塊使用白色底，整體配色以舒適、清楚為主。

### 資料編輯與更新

- 頁面提供「編輯資料」功能，按鈕放在「台中高中職」旁。
- 編輯資料需先輸入密碼 `22068816` 才能解鎖。
- 編輯畫面以接近 Excel 的方式呈現，方便直接修改資料。
- 使用本機 `server.py` 預覽時，編輯後的資料會寫回 `data/schools.json`。
- 使用 GitHub Pages 時，編輯後的資料只會存在目前瀏覽器中，不會自動寫回 GitHub。
- 頁面需顯示「上次更新」時間，時間來源為 `data/schools.json` 的 `updatedAt` 欄位；本機儲存時會同步更新。

### 品牌與版面

- 網站標題為「盛田教研｜國中落點地圖」。
- 頁面需顯示盛田教研 LOGO。
- 主視覺配色參考 LOGO 的深綠色，並搭配白色資訊卡片與灰色地圖遮罩。
- 版面分為左側資料操作區與右側地圖區。

## 本機預覽

```bash
python3 server.py
```

開啟：

```text
http://127.0.0.1:5173/
```

本機使用 `server.py` 時，表格編輯儲存會寫回 `data/schools.json`。

## 部署到 GitHub Pages

1. 在 GitHub 建立一個新的 repository。
2. 將這個 `school-map` 資料夾推上 GitHub。
3. 到 repository 的 `Settings` → `Pages`。
4. `Build and deployment` 選擇 `Deploy from a branch`。
5. Branch 選 `main`，資料夾選 `/root`，按 `Save`。
6. 等 GitHub Pages 部署完成後，網站會出現在：

```text
https://你的帳號.github.io/你的-repository-名稱/
```

## GitHub Pages 注意事項

GitHub Pages 只能提供靜態網站，不能執行 `server.py`，所以：

- 地圖、搜尋、學校資訊、密碼解鎖、表格編輯畫面都可以使用。
- 在 GitHub Pages 上編輯後，資料會儲存在目前瀏覽器，不會直接寫回 GitHub 上的 `data/schools.json`。
- 若要讓所有人打開都看到更新後的資料，需要把修改後的 `data/schools.json` 重新提交到 GitHub。

## 主要檔案

- `index.html`：網站頁面
- `styles.css`：版面與配色
- `app.js`：地圖、搜尋、資料顯示與編輯
- `data/schools.json`：學校與成績資料
- `data/taichung-boundary.json`：台中市邊界遮罩
- `assets/logo-avatar.png`：盛田教研 LOGO
- `server.py`：本機預覽與本機儲存用伺服器
